🧠 LLM (Large Language Model)

Your MediCare system uses Google Gemini 2.5 Flash - a powerful language model with 175+ billion parameters trained on medical knowledge.

How LLM Works in Your System

🙋 USER INPUT

"I have stomach ache since 2 days"

↓
🔤 TOKENIZATION

Text → Numbers: ["I", "have", "stomach", "ache"] → [123, 456, 789, 101]

↓
🧮 EMBEDDING

Numbers → High-dimensional vectors (512 dimensions per word)

↓
🎯 TRANSFORMER LAYERS (100+ layers)

Attention Mechanism: Understands context and relationships

"stomach" + "ache" + "2 days" → Likely Gastroenteritis

↓
📊 SOFTMAX PROBABILITY

Next word probabilities:

"How" = 45% | "That" = 30% | "Let" = 25%

↓
💬 OUTPUT GENERATION

"How long have you been experiencing vomiting?"

🎓 Key Concepts:
  • Transformer Architecture: Uses "attention mechanism" to understand context
  • Parameters: 175+ billion learned weights that encode medical knowledge
  • Pre-training: Trained on billions of medical documents, journals, websites
  • Fine-tuning: Specialized training for medical consultations

🌡️ Temperature & Parameters

Temperature (Randomness Control)

Controls how creative vs predictable the AI responses are

Example Output:

Balanced responses - Good for medical advice

🎯 Temperature = 0.1 - 0.3

Use for: Medical diagnoses, drug information

Effect: Very predictable, safe, factual

"Paracetamol 500mg every 6 hours" (same every time)

🎨 Temperature = 0.7 - 1.0

Use for: Health tips, general advice

Effect: Varied, creative suggestions

"Try ginger tea OR warm lemon water OR peppermint" (varies)

🎛️ Other Important Parameters

maxOutputTokens

What: Maximum response length

1 token ≈ 0.75 words

Recommended: 2048 tokens (≈1500 words)
Perfect for detailed treatment plans

topP (Nucleus Sampling)

What: Consider top P% of probable tokens

Range: 0.0 to 1.0

0.95: Consider top 95% most probable words
Diverse yet relevant responses

topK (Top-K Sampling)

What: Consider only top K tokens

Typical: 40-50

40: Only consider 40 most likely words
Prevents nonsense medical terms

stopSequences

What: Strings that stop generation

Use: Control response end

Example: ["}", "END"]
Stop after JSON closes

⚙️ Core Algorithms

1. Self-Attention Mechanism

Allows AI to understand relationships between all words in input:

Attention(Q, K, V) = Softmax(QKT / √dk) × V // Q = Query (what we're looking for) // K = Key (what's available) // V = Value (actual content)

Example: "fever" + "cough" + "3 days" → AI learns this pattern = respiratory infection

2. Softmax Function (Probability Distribution)

Converts neural network outputs to probabilities:

P(wordi) = e(scorei) / Σ e(scorej) // Example: Scores: "medicine" = 3.2, "vaccine" = 2.1, "surgery" = 0.5 After Softmax: "medicine" → 67% probability "vaccine" → 26% probability "surgery" → 7% probability
3. Beam Search (Text Generation)

Explores multiple response paths simultaneously:

Step 1: Generate 3 possible starts

  • "Based on symptoms, I recommend rest..."
  • "Based on symptoms, I recommend medicine..."
  • "Based on symptoms, I recommend testing..."

Step 2: Expand each path

Step 3: Pick path with highest total probability
