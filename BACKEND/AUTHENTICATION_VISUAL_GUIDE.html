🆕 User Registration Flow

1
User Opens register.html
Beautiful registration form with password strength meter
2
User Fills Form
Name, Email, Phone (optional), Password (min 6 chars)
3
POST /api/auth/register
Frontend sends user data to backend
4
Backend Validation
Check email doesn't exist, password length ≥ 6
5
Password Hashing
bcrypt.hash(password, 10) - pre-save hook
6
Save to MongoDB
User.create({ email, passwordHash, name, phone })
7
Generate JWT Token
jwt.sign({ userId, email }, SECRET, { expiresIn: '30d' })
8
Send Response
{ success: true, token, user: { id, email, name } }
9
Store in localStorage
localStorage.setItem('authToken', token)
10
Redirect to Welcome
User is now logged in! window.location.href = 'welcome.html'

🔑 User Login Flow

1
User Opens login.html
Email/password form with visibility toggle
2
Enter Credentials
Email: john@example.com, Password: securepass123
3
POST /api/auth/login
Send { email, password } to backend
4
Find User in Database
User.findOne({ email: email.toLowerCase() })
5
Verify Password
await user.comparePassword(password) - bcrypt.compare()
6
Update Last Login
user.lastLogin = new Date(); await user.save();
7
Generate New JWT Token
Fresh token with 30-day expiry
8
Send Response
{ success: true, token, user }
9
Store Token
localStorage.setItem('authToken', token)
10
Redirect to Welcome
Successfully logged in!

✨ Key Security Features

🔐 Password Security

  • Bcrypt hashing with 10 salt rounds
  • Never stored in plain text
  • Pre-save hook auto-hashes
  • Secure comparison with bcrypt.compare()
  • Minimum 6 characters required

🎫 JWT Token System

  • 30-day expiry (configurable)
  • Signed with secret key
  • Contains userId and email
  • Bearer token format
  • Auto-refresh on login

🛡️ Data Privacy

  • Each user sees ONLY their data
  • MongoDB queries filtered by userId
  • No cross-user data leakage
  • Protected API endpoints
  • Session expiry handling

🔄 Backward Compatibility

  • Old data without userId preserved
  • New analyses auto-link to user
  • Optional authentication supported
  • Anonymous usage still possible
  • Gradual migration path

💻 Code Examples

Frontend: Making Authenticated Requests

const token = localStorage.getItem('authToken'); const response = await fetch('http://localhost:5000/api/cough-analyses', { headers: { 'Authorization': `Bearer ${token}` } }); if (response.status === 401) { // Session expired - redirect to login localStorage.removeItem('authToken'); window.location.href = 'login.html'; }

Backend: Protecting Routes
